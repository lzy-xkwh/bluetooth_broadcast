# v1.6.7 更新说明

这版解决长时间待机后常驻通知消失、服务无法自行恢复的问题。

## 变化

- 定位原因：原版本只在开机、升级或打开 App 时启动服务，后续仅依赖 `START_STICKY`；部分厂商系统清理前台服务或通知后不会及时执行标准重启流程。
- 新增 15 分钟一次的轻量服务健康检查，长时间待机后可重新拉起服务并恢复常驻通知。
- 服务被正常销毁或最近任务被划掉时，额外安排一次 1 分钟后的快速恢复。
- 健康检查兼容 Android 深度待机，不持有唤醒锁，不持续占用 CPU。
- App 状态区新增“服务自恢复”及最近一次服务运行时间，方便真机判断保活是否工作。
- 记录最近一次健康检查及启动错误，为后续设备特定问题保留诊断依据。

## 使用提示

升级安装后请至少打开一次 App，让系统登记健康检查。自启动权限仍建议开启，电池策略建议设为“不限制”。手动“强行停止”会按 Android 规则暂停所有闹钟和接收器，需要再次打开 App 才能恢复。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

从 `v1.6.5` 开始，固定签名文件不再放在仓库里，而是通过 GitHub Secrets 注入。只要 Secrets 使用的仍是原来的 keystore，覆盖安装能力不变。
