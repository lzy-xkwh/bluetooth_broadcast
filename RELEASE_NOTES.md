# v1.6.6 更新说明

这版修复兼容模式保存、开机自启动和自动化通知通用化问题。

## 变化

- 修复进入 App 加载配置时，蓝牙兼容模式可能被提前保存为未勾选的问题，并对旧版本误写状态做一次性恢复。
- 蓝牙兼容模式默认开启；关闭后为高强度广播，文案明确提示蓝牙异常、耗电和干扰风险。
- 新增开机启动接收器，系统重启或 App 升级后自动拉起常驻通知服务。
- 自动化通知文案改为通用车主 App 场景，不再绑定特定品牌。
- 通知监听服务在系统设置里的显示名改为“桩小易自动化通知解锁”。
- 保留原通知监听组件类名，避免已授权用户因组件名变化被迫重新授权。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

从 `v1.6.5` 开始，固定签名文件不再放在仓库里，而是通过 GitHub Secrets 注入。只要 Secrets 使用的仍是原来的 keystore，覆盖安装能力不变。
