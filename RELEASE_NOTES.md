# v1.6.5 更新说明

这版集中修复代码审查发现的 6 个主要问题，重点是蓝牙稳定性、触发准确性、签名安全和代码职责拆分。

## 变化

- 新增“蓝牙兼容模式”，默认使用 `BALANCED + MEDIUM` 广播参数，降低手机蓝牙异常提示概率。
- 车载蓝牙连接自动触发新增 60 秒冷却，避免重复连接事件反复广播。
- 小鹏通知自动触发新增可选来源包名过滤，降低其他通知误触发概率。
- App 内“刷新通知快捷按钮”改名为“刷新常驻通知”，与当前通知栏职责一致。
- Release 签名改为 GitHub Secrets 注入，keystore 不再提交到仓库。
- 将 UUID、加密、CRC、iBeacon 数据生成逻辑从 `MainActivity` 拆到 `BeaconProtocol`。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

从 `v1.6.5` 开始，固定签名文件不再放在仓库里，而是通过 GitHub Secrets 注入。只要 Secrets 使用的仍是原来的 keystore，覆盖安装能力不变。
