# v1.6.9 更新说明

这版降低后台健康检查频率，并修复现代 Android 权限和自动触发链路中的稳定性问题。

## 变化

- 常规服务健康检查由 15 分钟调整为 6 小时，每天最多 4 次；服务销毁后的 1 分钟快速恢复和所有事件触发恢复保持不变。
- 修复 Android 12 及以上未授予附近设备权限时，蓝牙能力检测可能抛出权限异常的问题。
- 增加 Android 13 及以上通知显示权限声明、请求和状态提示，避免常驻服务运行但通知栏不可见。
- 通知监听服务重新连接时主动恢复常驻服务和下一次健康检查。
- 自动化通知忽略本 App 自己发布的通知，避免用户设置过于宽泛的关键词后自触发。
- 自动化通知和车载蓝牙触发只有在后台服务启动请求成功后才进入 60 秒冷却；启动失败会保留诊断记录。
- 修复系统时间向后调整时，持久化冷却时间可能长时间阻止自动触发的问题。
- 车载蓝牙广播接收器改为不对普通第三方 App 导出，降低伪造广播触发风险。
- PendingIntent 增加不可变标志，移除未使用的明文网络开关，并补充无效 Key 的明确提示。
- README 修正已移除的通知栏按钮、旧“车机”文案和健康检查频率描述。
- 修复带 Key 的加密设备点击“停止充电”时实际生成解锁包的风险：由于原小程序没有已验证的加密停充载荷，现在会明确取消操作；无 Key 设备不受影响。

## 使用提示

升级安装后请至少打开一次 App，授予系统请求的蓝牙和通知显示权限，并让系统登记健康检查。自启动权限仍建议开启，电池策略建议设为“不限制”。手动“强行停止”会按 Android 规则暂停所有闹钟和接收器，需要再次打开 App 才能恢复。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

从 `v1.6.5` 开始，固定签名文件不再放在仓库里，而是通过 GitHub Secrets 注入。只要 Secrets 使用的仍是原来的 keystore，覆盖安装能力不变。
