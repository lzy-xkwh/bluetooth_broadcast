# v1.5.2 更新说明

这版继续优化通知栏视觉协调性，同时保留固定签名和配置确认改进。

## 变化

- 通知栏折叠状态右侧显示 `待命 ▼` 或 `广播中 ▼`，提示可以下拉展开查看完整内容。
- 通知布局改得更接近系统通知列表，展开态按钮从大色块改为轻量文字操作。
- 自定义通知内容里的图标改为和 App 图标一致，并重新设计为更简洁的圆角方块图标。
- 状态栏小图标改为 Android 要求的白色剪影图标，避免系统状态栏显示异常。
- 更换 App 图标：蓝牙、钥匙、充电启动三个元素更清晰。
- 增加 `保存配置` 按钮。
- 增加 `已保存 MAC` 摘要，保存后可以直接确认当前生效的 MAC、Key 状态和设备类型。
- Release 构建改用固定自用签名 keystore。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

如果手机上已经安装的是 `v1.5.0` 或更早版本，由于旧版 GitHub Actions 每次构建会生成随机 debug 签名，第一次升级到 `v1.5.1` 时 Android 仍可能提示签名不一致。这种情况需要先卸载旧版，再安装 `v1.5.1`。之后再升级新版本就不需要卸载了。
