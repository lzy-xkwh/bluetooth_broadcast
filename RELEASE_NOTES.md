# v1.6.4 更新说明

这版统一 App 内按钮、小组件和自动化触发的广播入口。

## 变化

- App 内点击 `解锁/启动充电`、`停止充电`、`停止广播` 时，改为发送后台服务 action。
- BLE 广播只由 `KeyBroadcastService` 管理，避免 Activity 和 Service 各自持有一套 advertiser。
- 通知栏状态会随 App 内按钮操作同步变化，例如 `广播中`、`待命`。
- App 页面保留配置校验和保存，不再直接启动 BLE 广播。

## 关于覆盖安装

从 `v1.5.1` 开始，GitHub Release 里的 APK 使用同一把固定签名 key，后续版本可以直接覆盖安装升级。

如果手机上已经安装的是 `v1.5.0` 或更早版本，由于旧版 GitHub Actions 每次构建会生成随机 debug 签名，第一次升级到 `v1.5.1` 时 Android 仍可能提示签名不一致。这种情况需要先卸载旧版，再安装 `v1.5.1`。之后再升级新版本就不需要卸载了。
