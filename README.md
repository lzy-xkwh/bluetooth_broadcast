# 桩小易蓝牙钥匙

这是一个不依赖微信、不依赖小程序运行时的 Android 原生极简 App，用来模拟原小程序的蓝牙 iBeacon 解锁/启动充电广播。

当前版本重点面向手机使用：可以手动点击、桌面小组件、车主 App 自动化通知触发。

## APK 信息

- 输出文件：`build/apk/poersmart-car-key.apk`
- 包名：`com.poersmart.charge`
- 应用名：`桩小易蓝牙钥匙`
- 当前版本：`1.6.7-car-key`
- SHA256：见 `build/apk/poersmart-car-key.apk.sha256`
- 签名：Release 使用 GitHub Secrets 中的固定签名；本地未配置时会生成开发签名

## 已实现功能

- 保存充电桩 MAC。
- 可选保存 32 位十六进制 Key。
- 复刻原小程序 `utils/beacon_api.js` 的 UUID 生成逻辑。
- 使用 Android 原生 `BluetoothLeAdvertiser.startAdvertising()` 广播 iBeacon。
- “解锁/启动充电”广播 10 秒后自动停止。
- “停止充电”广播 10 秒后自动停止。
- App 按钮、桌面小组件、自动化通知触发统一走同一个后台服务，通知栏状态会同步变化。
- 常驻通知栏只显示当前状态和操作摘要，不再承载按钮。
- 桌面小组件提供主要操作按钮：`解锁 / 停充 / 停止`，点击标题进入设置。
- 支持车主 App 自动化通知触发：收到包含 `POER_UNLOCK_CHARGER` 的通知后自动广播。
- 支持可选来源 App 包名过滤，降低其他通知误触发概率。
- 支持车载蓝牙连接触发，但不建议作为主要自动化方案。
- 车载蓝牙连接触发内置 60 秒冷却，避免重复连接事件反复广播。
- 默认启用蓝牙兼容模式，使用较温和的 BLE 广播参数；需要更远距离时可关闭。
- 首屏优先显示状态和操作按钮，充电桩配置下移为设置区。
- 进入 App 不再自动聚焦 MAC 输入框，不会主动弹出键盘。
- 内置蓝牙钥匙/充电启动主题图标。
- 提供“保存配置”按钮和“已保存 MAC”摘要，明确确认当前生效配置。
- Release 构建使用 GitHub Secrets 提供的固定自用签名；从固定签名版开始，后续版本可覆盖安装升级。

## 推荐使用方式

推荐主流程是：

1. 车主 App 检测到插入充电枪。
2. 车主 App 自动化发送手机通知。
3. 通知内容包含 `POER_UNLOCK_CHARGER`。
4. 本 App 的通知监听服务识别关键词。
5. 本 App 自动广播 10 秒解锁/启动充电 iBeacon。

这样触发点接近真实充电动作，比“连接车载蓝牙时自动解锁”更准确。

## 首次安装配置

### 1. 安装 APK

安装：

```bash
adb install -r /root/wxre/wx_standalone_apk/build/apk/poersmart-car-key.apk
```

也可以把 APK 传到手机后手动安装。

### 2. 打开 App

打开“桩小易蓝牙钥匙”。

首页顶部是“状态与操作”：

- `解锁/启动充电`：立即广播 10 秒。
- `停止充电`：广播停充指令 10 秒。
- `停止广播`：立即停止当前广播。
- `检测蓝牙广播能力`：检查当前设备是否支持 BLE advertiser。
- `刷新常驻通知`：保存当前配置并刷新通知栏状态。

充电桩 MAC、Key、设备类型在下方“充电桩信息”区域维护。进入 App 时不会自动弹出输入框，需要修改时再点对应输入项。
修改充电桩信息后，点击 `保存配置`，页面会显示 `已保存 MAC：...` 用来确认当前保存值。

### 3. 填写充电桩信息

在“充电桩信息”区域填写：

- `MAC`：充电桩 MAC，支持带冒号或不带冒号，保存时会自动规范为 12 位十六进制。
- `Key`：可选，32 位十六进制；没有就留空。
- `设备类型`：默认选“普通/交流桩”，如果你确认是直流桩再切换。
- `蓝牙兼容模式`：默认开启，使用 `BALANCED + MEDIUM` 广播参数，优先降低手机蓝牙异常提示。
- 关闭 `蓝牙兼容模式` 后会启用高强度广播，距离可能更远，但可能增加蓝牙异常提示、耗电和对其他蓝牙连接的干扰风险。
- 从旧版本升级后，App 会自动把旧版本误保存的高强度状态恢复为兼容模式；之后如果你手动关闭兼容模式，会按你的选择保存。

填写后点击一次“保存配置”，页面会显示已保存的 MAC、Key 状态、设备类型和蓝牙模式。需要刷新通知栏状态时，再点击“刷新常驻通知”。

### 4. 检查蓝牙广播能力

点击“检测车机蓝牙广播能力”。

看到“可用：车机支持 BLE 广播”才可以正常广播。

如果提示不可用，常见原因是：

- 蓝牙未开启。
- 手机/车机不支持 BLE advertiser。
- Android 12 及以上没有授权蓝牙相关权限。
- 系统省电策略限制后台蓝牙。
- 如果其他 App 提示手机蓝牙异常，优先确认“蓝牙兼容模式”已开启。

## Release 签名配置

GitHub Release 构建不再提交 keystore 文件到仓库。需要在仓库 Secrets 中配置：

- `SIGNING_KEYSTORE_B64`：keystore 文件的 Base64 内容。
- `SIGNING_KEY_ALIAS`：key alias，例如 `poersmart`。
- `SIGNING_KEYSTORE_PASS`：keystore 密码。
- `SIGNING_KEY_PASS`：key 密码。

本地执行 `./build_minimal.sh` 时，如果没有配置签名 Secret，会在 `build/signing/` 下生成开发签名，适合本地测试；GitHub Release 构建会要求固定签名 Secret，缺失时直接失败，避免发布随机签名 APK。

## 自动化通知解锁

这是推荐的无感方案。

### 1. 在车主 App 创建自动化

在任意支持自动化、智慧场景或条件通知的车主 App 中创建规则：

- 触发条件：插入充电枪、进入充电准备、充电枪连接，选择你手机上实际可用的条件。
- 执行动作：发送手机通知。
- 通知内容：`POER_UNLOCK_CHARGER`

关键词必须出现在系统通知的标题或正文里。只在车主 App 内部消息中心出现、不进入系统通知栏的消息无法被监听。

### 2. 在本 App 开启监听

打开“桩小易蓝牙钥匙”：

1. 确认“自动化通知关键词自动解锁”已勾选。
2. 确认“自动化通知关键词”为 `POER_UNLOCK_CHARGER`。
3. 可选填写“通知来源包名”，留空表示只按关键词匹配。
4. 点击“打开通知使用权设置”。
5. 找到“桩小易自动化通知解锁”。
6. 授权通知使用权。

授权后回到 App，状态区会显示：

```text
通知监听：已授权
```

### 3. 触发效果

锁屏或亮屏时，只要系统收到包含 `POER_UNLOCK_CHARGER` 的通知，本 App 会自动执行“解锁/启动充电”广播。

为避免车主 App 同一事件重复推送导致反复广播，App 内置了 60 秒冷却时间：同一关键词 60 秒内只会触发一次。

如果担心其他 App 通知误触发，可以在“通知来源包名”里填写目标车主 App 的实际包名。不同品牌和不同版本的包名可能不同，不确定时先留空，确认后再填写。

## 通知栏快捷按钮

App 启动后会创建常驻通知。通知栏只显示一行轻量状态，不再放操作按钮：

- 左侧：App 图标，尺寸按普通通知收敛。
- 中间：当前操作摘要，例如 `小组件可解锁/停充/停止`。
- 右侧：状态短标签，例如 `待命`、`广播中`、`不可用`。

通知栏显示已做可读性优化：

- 使用自定义通知布局，显式指定深色文字和浅色背景，避免部分国产系统主题把文字渲染成浅色。
- 固定为折叠态单行状态展示，不再设置展开通知样式。
- 解锁、停充、停止操作统一放到桌面小组件。

如果通知栏仍然看不到完整内容，先尝试下拉展开通知。部分手机系统会把常驻通知压缩到“静默通知”区域，可以在系统通知设置里把本 App 通知调整为允许显示或非静默。

## 桌面小组件

如果手机桌面支持小组件，可以添加“桩小易蓝牙钥匙”小组件。

小组件包含：

- `解锁`
- `停充`
- `停止`

点击小组件标题区域进入 App 设置。小组件采用紧凑控制面板布局，适合不打开 App 时直接操作。

## 车载蓝牙自动解锁

这个功能仍然保留，但不推荐作为主要方案。

使用方法：

1. 勾选“连接车载蓝牙时自动解锁”。
2. 在“自动触发条件”里填写车载蓝牙名称或 MAC 地址的一部分。
3. 点击“刷新常驻通知”保存。

限制：

- 蓝牙连接通常发生在刚上车时，不一定是准备充电。
- 下车插枪时可能已经错过触发时机。
- “自动触发条件”留空会在任意蓝牙设备连接时触发，不建议长期这样用。

## 开机自启动

App 已注册开机启动接收器。手机重启或 App 升级后，系统会尝试自动拉起常驻通知服务。

App 还会每 15 分钟安排一次轻量服务健康检查。即使厂商系统在长时间待机后撤掉前台服务或常驻通知，下一次检查也会重新拉起服务并恢复通知。检查由系统闹钟调度，不持续占用 CPU，不持有唤醒锁；深度省电状态下，实际执行时间可能被系统延后。

打开 App 后，状态区会显示“服务自恢复：已启用”和最近一次服务运行时间，可用于确认恢复机制是否仍在执行。

如果手机系统有“自启动管理”“后台启动”“允许关联启动”等权限，需要把本 App 加入允许列表。不同系统策略不同，即使 App 已注册开机启动，系统也可能在未授权时拦截启动。

## 日常使用建议

推荐组合：

1. 主触发：车主 App 自动化通知。
2. 备用触发：通知栏“解锁”按钮。
3. 手动兜底：打开 App 点击“解锁/启动充电”。
4. 不需要时：关闭“连接车载蓝牙时自动解锁”。

安全建议：

- 自动化通知关键词不要设置成普通中文句子，避免误触发。
- 推荐继续使用 `POER_UNLOCK_CHARGER` 这种唯一文本。
- 不建议长时间持续广播。
- 不建议开启“任意蓝牙连接都触发”。

## 常见问题

### 收到自动化通知后没有自动解锁

按顺序检查：

1. 通知内容是否真的包含 `POER_UNLOCK_CHARGER`。
2. 系统通知栏是否能看到这条通知。
3. 本 App 状态区是否显示“通知监听：已授权”。
4. “自动化通知关键词自动解锁”是否勾选。
5. MAC 是否已保存且为 12 位十六进制。
6. 手机蓝牙是否开启。
7. 是否刚刚触发过一次，60 秒冷却期内不会重复触发。

### 锁屏时没有触发

车主 App 自动化通知必须能在锁屏时进入系统通知栏。

检查：

- 车主 App 通知权限是否打开。
- 车主 App 通知是否允许锁屏显示。
- 手机是否开启了省电模式、睡眠待机优化或限制后台通知。
- 本 App 是否被系统限制后台运行。

### 通知栏文字看不清

新版通知已经改为自定义布局，文字颜色固定为深色，背景固定为浅色，避免系统通知模板在部分主题下文字反色异常。

如果仍然看不清：

- 下拉展开通知查看大文本。
- 在系统设置里把本 App 通知设为正常显示，而不是静默折叠。
- 避免使用系统的“隐藏通知内容”或极简通知模式。

### 通知栏使用一段时间后消失

`v1.6.7` 起已增加 15 分钟服务健康检查。原版本只依赖前台服务的 `START_STICKY` 和开机自启动；部分厂商系统长时间待机后会撤掉服务或通知，却不按 Android 标准及时重启服务，因此通知可能永久消失到下次打开 App。

升级后先打开一次 App，确认状态区显示“服务自恢复：已启用”。如果通知消失，通常会在下一次健康检查时恢复；深度省电可能延后检查。若一直不恢复，请同时检查：

- App 的自启动、后台启动和关联启动是否允许。
- 电池策略是否设为“不限制”或“允许后台活动”。
- 系统通知设置是否允许显示本 App 的常驻通知。
- 是否手动“强行停止”过 App；强行停止后 Android 会暂停闹钟和接收器，必须手动打开一次 App 才能恢复。

### 点击解锁后充电桩没有反应

检查：

- MAC 是否正确。
- Key 是否需要填写；如果原小程序绑定设备用了 Key，就要填 32 位 Key。
- 设备类型是否选对。
- 手机是否支持 BLE 广播。
- 手机和充电桩距离是否过远。
- 原小程序同样操作是否仍可解锁，用来排除充电桩状态问题。

### 提示手机不支持 BLE 广播

本 App 依赖 Android BLE Peripheral/Advertiser 能力。

如果 `BluetoothAdapter.isMultipleAdvertisementSupported()` 为 false，这台设备无法模拟小程序蓝牙广播。普通蓝牙连接能力不等于 BLE 广播能力。

## 权限说明

本 App 可能使用以下权限：

- 蓝牙权限：用于发起 BLE iBeacon 广播。
- 前台服务：用于保持通知栏快捷按钮和短时广播服务。
- 通知使用权：仅用于读取系统通知标题/正文，识别 `POER_UNLOCK_CHARGER` 关键词。

通知监听逻辑只做关键词匹配，不上传数据，不依赖网络。
